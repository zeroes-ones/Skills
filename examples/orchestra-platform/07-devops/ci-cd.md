# CI/CD Pipeline — Orchestra Platform

## Pipeline Stages

Orchestra uses **GitHub Actions** with the following pipeline:

```
lint → test (unit + integration) → build → container-scan → deploy-staging → e2e-test → deploy-prod (approval gate)
```

| Stage | What It Does | Pass Criteria |
|---|---|---|
| **lint** | ESLint, Prettier, ShellCheck, Terraform fmt | Zero warnings |
| **test** | Jest (unit) + Playwright (integration) | All pass, coverage > 80% |
| **build** | Docker multi-stage builds per service, Turborepo | All services build clean |
| **container-scan** | Trivy vulnerability scan, Dockle best practices | No HIGH/CRITICAL CVEs |
| **deploy-staging** | Helm upgrade via ArgoCD sync | Healthy rollout, no crash loops |
| **e2e-test** | Playwright end-to-end critical paths | All pass, Lighthouse > 90 |
| **deploy-prod** | Manual approval → Helm upgrade → smoke tests | Approval + green smoke tests |

## Build Optimization

- **Matrix builds** — services build in parallel across a 4-runner matrix, sharded by service dependency graph.
- **Docker layer caching** — `cache-from` and `cache-to` with ECR as the cache backend. Base layers rarely rebuild; application layers rebuild on code change.
- **Turborepo remote caching** — shared build cache across CI runners with a **73% cache hit rate**, cutting average build time from 8 minutes to 2 minutes.

## Quality Gates

All gates are enforced in the pipeline — merge is blocked if any fail:

- Test coverage below **80%** (measured by NYC/Istanbul)
- Any **HIGH or CRITICAL** CVE in container images
- JavaScript bundle size exceeds **500 KB** (compressed, per route chunk)
- Lighthouse performance and accessibility scores below **90**
- TypeScript strict mode violations

## Supply Chain Security — SLSA Level 3

Orchestra achieves **SLSA Build Level 3**:

- **Hermetic builds** — all dependencies are pinned (lockfiles committed), no network access during build except to the package registry.
- **Signed provenance attestations** — each container image includes an in-toto attestation generated by the GitHub Actions workflow, verifiable with `cosign verify-attestation`.
- **Verified artifact integrity** — deployments verify attestations before rollout. Images without valid attestations are rejected at the admission controller.

## Release Branches & Rollback

- **`release/v1.x`** — long-lived release branches receive backported fixes.
- **`hotfix/*`** — triggers an expedited pipeline that skips integration tests (unit + smoke only) for speed.
- **Rollback** — `helm rollback` invoked via ArgoCD UI or CLI. Median rollback time is under 5 minutes. Automated rollback triggers when the error budget drops below 20% remaining — no human approval needed in that scenario.
