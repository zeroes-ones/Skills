# Exploration Checklist: 9-Step Verification

## Phase 1: Problem Framing
1. [ ] **Problem Statement:** One sentence. Cites evidence, not assumptions.
   - Red flag: "Everyone needs this" — no product is for everyone.
2. [ ] **Success Criteria:** Measurable metric with baseline and target.
   - Red flag: "Better UX" — not measurable.
3. [ ] **Persona:** Named, specific. Current behavior and pain point quantified.
   - Red flag: "Power users" — not a persona.
4. [ ] **MVP Scope:** Single deliverable. What is the ONE thing we ship first?
   - Red flag: "We need all of it" — then you need to learn, not build.
5. [ ] **Constraints:** Every constraint documented with origin story.
   - Red flag: "No constraints" — there are always constraints.

## Phase 2: Solution Space Exploration
6. [ ] **Approaches:** At least 3 distinct approaches. One must be the opposite of instinct.
   - Red flag: All approaches are variations of the same idea.
7. [ ] **Trade-Offs:** Per approach, what we GAIN and what we LOSE.
   - Red flag: An approach is described as having "no downsides."
8. [ ] **Uncertainties:** Known-unknowns listed with resolution plan.
   - Red flag: "We're confident" on every dimension.

## Phase 3: Gate
9. [ ] **Spec Review Gate:** Design brief produced. Explicit approval decision.
   - Red flag: "Let's start coding and see what happens."
