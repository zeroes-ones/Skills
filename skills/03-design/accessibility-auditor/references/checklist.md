# Production Checklist

<!-- QUICK: 30s -- all items are machine-verifiable. Every item gets a validation command and auto-fix path. -->

| ID | Checklist Item | Validation Command | Auto-Fix |
|----|---------------|-------------------|----------|
| **S1** | WCAG 2.2 conformance target (A/AA/AAA) defined and communicated to all stakeholders | `grep -cP 'WCAG.*2\.[0-2].*[A]{1,3}\|conformance.*target.*[A]{1,3}' a11y-plan.md` — must match | Add `## Conformance Target: WCAG 2.2 [Level]` to audit plan. Template: `templates/a11y-conformance-target.md` |
| **S2** | Automated testing (axe-core + Lighthouse) running in CI with quality gates (≥ 95 Lighthouse a11y, 0 axe violations) | `npx @axe-core/cli <url> --rules all` returns 0 violations; `npx lighthouse <url> --only-categories=accessibility` score ≥ 95 | CI config: `.github/workflows/a11y.yml` with `npx axe-core <url> --exit && npx lighthouse <url> --only-categories=accessibility --chrome-flags="--headless"` — fails build on violations |
| **S3** | Manual keyboard testing completed on all critical user flows | `grep -cP '(keyboard.*test.*pass\|Tab.*order.*verified\|keyboard.*flow.*complete)' manual-test-results.md` — must return >= number of critical flows | Keyboard test script: `scripts/keyboard-audit.sh` — walks through each flow with Tab, Shift+Tab, Enter, Escape, Space, Arrow keys |
| **S4** | Screen reader testing completed: VoiceOver (macOS/Safari) + NVDA (Windows/Chrome) on core flows | `grep -cP '(VoiceOver.*tested\|NVDA.*tested\|screen.reader.*pass)' manual-test-results.md` — must show both VoiceOver AND NVDA | Screen reader test script: `scripts/sr-audit.sh` — runs VoiceOver test flow on macOS, NVDA test flow checklist for Windows |
| **S5** | Semantic HTML audit: single `<main>`, named `<nav>` elements, heading hierarchy without skips | `npx html-validate --rule heading-level --rule landmark-no-duplicate <url>` — 0 errors | Run `scripts/semantic-audit.sh` — checks: 1 `<main>`, 1 `<h1>`, named `<nav>` with `aria-label`, no heading level gaps |
| **S6** | All interactive elements have visible focus indicators (no `outline: none` without custom indicator) | `grep -rn 'outline\s*:\s*none\|outline\s*:\s*0' *.css *.scss \| wc -l` — if > 0, validate each has custom `box-shadow` or `outline-offset` within 5 lines | CSS lint: add `stylelint-a11y` with `a11y/focusable-selectors` rule. Run `npx stylelint '**/*.css' --rule 'a11y/focusable-selectors: true'` in CI |
| **S7** | Focus trapping implemented and tested in all modals, drawers, and dialogs | `grep -rn 'role="dialog"\|role="alertdialog"' *.html *.tsx *.jsx \| grep -v 'aria-modal\|focus-trap\|onKeyDown\|Escape'` — must return 0 (every dialog has trap logic) | Run `scripts/check-modal-a11y.sh` — validates every dialog has: `aria-modal`, Escape handler, focus trap, focus restore on close |
| **S8** | All form inputs have associated labels; errors linked via `aria-describedby`; `aria-invalid` on error | `npx axe-core <url> --rules label,label-title-only,aria-input-field-name,aria-valid-attr-value` — 0 violations | Run `scripts/check-forms.sh` — verifies: every `<input>` has `<label>` or `aria-label`, errors have `aria-describedby` + `aria-invalid`, required fields marked with `required` + `aria-required` |
| **S9** | Color is never the sole differentiator for information (supplement with icons, text, or patterns) | `npx axe-core <url> --rule color-contrast` — 0 violations. Manual: `scripts/check-color-only-info.sh` inspects for red/green only indicators without icons or text | Design system rule: every status indicator must have text + color (not color alone). All charts use patterns + colors. Error/success states always include icon + text |
| **S10** | Accessibility statement published with conformance claims, known limitations, and feedback contact info | `curl -s -o /dev/null -w '%{http_code}' <url>/accessibility` — must return 200. `grep -cP '(conformance.*claim\|known.*limitation\|feedback\|contact)' accessibility.html` — >= 3 matches | Template: `templates/accessibility-statement.html` — auto-generates from audit results: conformance level, criteria coverage %, known gaps, contact email, last review date |
