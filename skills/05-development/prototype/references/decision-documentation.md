# Decision Documentation: Prototype Results to Actionable Decisions

## Document Template
```markdown
# Prototype Decision: [Topic]
**Date:** YYYY-MM-DD
**Prototyper:** [Name]

## Design Question
[One sentence. The decision this prototype informs.]

## Hypothesis
We believe [Approach X] will [expected outcome] because [reasoning].
The prototype disproves this if [falsifiable condition].

## Prototype Approach
[Brief description of what was built. 2-3 sentences.]

## Results
[HYPOTHESIS CONFIRMED / DISPROVED / INCONCLUSIVE]
[Evidence: specific numbers, observed behavior, surprises.]

## Evidence Quality: [HIGH / MEDIUM / LOW]
[Rationale for rating.]

## Decision
[The decision made based on these results.]

## Prototype Shortcuts
[Implementation shortcuts that differ from production intent.]
```

## Archive Structure
Store decision documents in: `docs/decisions/YYYY-MM-DD-[topic]-prototype-result.md`
This creates an organizational knowledge base of prototype-informed decisions.
