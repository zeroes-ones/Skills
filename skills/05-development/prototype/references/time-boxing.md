# Time Boxing: The 20-Minute Prototype

## The 20-Minute Rule
Research shows prototype value plateaus at 20 minutes. Beyond 20 minutes:
- Additional time produces diminishing returns on decision quality.
- The prototype transitions from "experiment" to "building."
- Emotional attachment to the code increases, making disposal harder.

## Timer Protocol
1. **00:00** — Start timer. Begin coding immediately.
2. **10:00** — Checkpoint. Are you testing the hypothesis or building infrastructure? If building infra, return to hypothesis.
3. **18:00** — Feature freeze. STOP adding code. Test the hypothesis with what you have.
4. **20:00** — HARD STOP. Cease all activity. Document results.

## When the Timer Expires
- Hypothesis confirmed/disproved → document and dispose.
- Inconclusive → document why. The question was too broad. Split and schedule next prototype.
- Halfway through → you scoped too broadly. Split the question into smaller units.

## The Psychology of the Hard Stop
The hard stop is the most important part of time-boxing. Without it, "just 5 more minutes" becomes an hour. The discipline of stopping is what separates prototyping from building.
