# Anti-Patterns

<!-- MACHINE-EXECUTABLE: Each row has a grep/lint pattern for detection and auto-prevention -->

| ❌ Anti-Pattern | ✅ Do This Instead | 🔍 Detect (grep/lint) | 🛡️ Auto-Prevent |
|-----------------|---------------------|--------------------------|-------------------|
| Optimizing detection models for recall without measuring false positive rates | Target <0.1% false positive rate; run new models in shadow mode for 2+ weeks; sample 5% of automated decisions for human audit; every successful appeal is a training example | `grep -rn "recall\|maximize.recall\|catch.all" model_config.yaml` when `grep -rn "false.positive\|precision\|FPR" model_config.yaml` → 0 matches = fail | **Precision-gate lint**: CI rule `npx validate-model-config model_config.yaml --require-precision-metric --min-precision 0.999 --severity high` |
| Using the same precision-recall trade-off for CSAM detection as for spam detection | Prioritize precision over recall for high-severity categories (CSAM, self-harm): a false positive CSAM flag is a life-destroying label; detection is triage, not decision-making | `grep -rn "CSAM\|self.harm\|suicide" classifier_config.yaml` when threshold same as `spam\|low.severity` = flag | **Severity-calibrated lint**: CI rule `npx validate-classifier-thresholds --per-severity --require "CSAM:precision>0.999\|self_harm:precision>0.999"` |
| Treating moderator wellness as an HR perk rather than an operational requirement | Mandatory exposure time limits (max 2 hours/day on graphic queues), content rotation, wellness check-ins, counseling access, and exit pathways with no career penalty | `grep -rn "moderator.wellness\|exposure.limit\|content.rotation\|trauma.support\|counseling" moderator_program.md` → matches < 5 = fail | **Moderator safety lint**: CI rule `npx validate-moderator-program moderator_program.md --require-wellness --min-elements 5` |
| Starting evidence chain of custody only when legal hold is triggered | Cryptographic chaining and WORM storage from moment of detection; log every access, action, and timestamp; chain of custody that starts late is not defensible in court | `grep -rn "legal.hold\|preservation.request" when NOT `crypto.hash\|WORM\|chain.of.custody\|RFC.3161` at detection time = flag | **Chain-of-custody lint**: CI rule `npx validate-evidence-pipeline --require-custody-from-detection --hash-algorithm SHA-256` |
| Rejecting appeals automatically or requiring full account access to submit an appeal | Every enforcement action (including permanent bans) must have human-reviewed appeal; appeal form must be accessible in limited-access mode for suspended users | `grep -rn "appeal.*auto\|auto.reject.*appeal\|appeal.*denied" appeal_config.yaml` → matches = block AND `grep -rn "limited.access\|suspended.access" appeal_config.yaml` → 0 = fail | **Appeal-access lint**: CI rule `npx validate-appeal-config appeal_config.yaml --require-human-review --require-limited-access-mode` |
| Sharing raw user data in cross-platform threat intelligence | Share only hashed identifiers (email hashes, device fingerprint hashes, content hashes) via automated APIs; never share raw PII in ad-hoc emails | `grep -rn "share.*email\|share.*phone\|share.*IP\|export.*user" threat_intel.yaml` → matches without `hash\|SHA.256` = block | **Intel-sharing lint**: CI rule `npx validate-threat-intel-sharing threat_intel.yaml --require-hashed-only --forbid-raw-pii` |
| Assuming English keyword lists work for all supported languages | Build language-specific keyword lexicons with native speakers; validate on real-world abusive content; measure false positives per language independently | `grep -rn "language.*support\|multilingual" detection_config.yaml` when `grep -rn "native.speaker\|per.language.validation\|language.specific.lexicon" detection_config.yaml` → 0 = fail | **Multilingual lint**: CI rule `npx validate-multilingual-detection --require-native-speaker-lexicon --per-language-fpr` |
| Neglecting penetration testing of reporting infrastructure | Quarterly penetration testing of full report → triage → action → appeal pipeline; test false report floods, weaponized reports targeting innocent users, and appeal system DoS | `grep -rn "penetration.test\|pentest\|security.test.*reporting" test_plan.yaml` → 0 matches = fail | **Reporting pentest gate**: CI rule `npx validate-test-plan test_plan.yaml --require-reporting-pentest --frequency quarterly` |
