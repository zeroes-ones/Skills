|---|---|---|

| **SBOM Lifecycle** | SPDX 3.0 vs CycloneDX 1.6 format selection, VEX integration, SBOM diffing, NTIA minimum elements | When designing SBOM generation and consumption pipelines |
| **Dependency Security** | Typosquatting detection, dependency confusion defense, slopsquatting in ML pipelines, malicious package analysis | When investigating a suspicious package or designing dependency firewall rules |
| **Artifact Signing** | Cosign keyless signing, Sigstore Fulcio/Rekor, container image signing, package provenance | When implementing signing for a specific artifact type or registry |
| **CI/CD Hardening** | Pipeline security review, OIDC federation, runner isolation, secret scanning, branch protection | When reviewing CI/CD pipeline configurations for security gaps |
| **Vendor Assessment** | Third-party SBOM collection, vendor security posture evaluation, contractual attestation requirements | When onboarding a new vendor that supplies software components or services |
