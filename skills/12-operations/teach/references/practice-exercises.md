# Practice Exercises

## Exercise Design Principle

Practice is the learning, not the assessment. Exercises are not tests — they are the mechanism by which understanding is built. The goal is productive struggle, not perfect performance.

## Difficulty Progression

Every concept session includes 3 exercises at increasing distance from the worked example:

### Level 1: Near-Transfer (Easy)
- Slightly modified version of the worked example
- Same pattern, different data
- Purpose: Build confidence, establish the pattern
- Time: 3-5 minutes

### Level 2: Application (Medium)
- Same concept, different context
- Requires combining the concept with ONE previously learned concept
- Purpose: Solidify understanding through variation
- Time: 5-10 minutes

### Level 3: Far-Transfer (Hard)
- Novel problem requiring the concept
- May require combining with 2+ previous concepts
- Purpose: Test generalization, reveal gaps
- Time: 10-15 minutes

## Exercise Template

```markdown
## Exercise [N]: [Title]
**Difficulty:** [Easy | Medium | Hard]
**Time:** [X] minutes

### Task
[Clear, unambiguous problem statement]

### Starter Code (if applicable)
```[language]
[Minimal starting point]
```

### Expected Behavior
- Input: [example input]
- Output: [expected output]

### Stretch Goal (optional)
[Extension for fast finishers]
```

## Exercise Design Checklist

- [ ] Problem statement is unambiguous (no hidden assumptions)
- [ ] Can be completed in the stated time
- [ ] Tests a specific concept (not a grab-bag)
- [ ] Starter code reduces boilerplate without giving away the solution
- [ ] Expected behavior is clearly specified
- [ ] Stretch goal exists for fast finishers (prevents idle time)

## When to Intervene

| Signal | Action |
|--------|--------|
| No progress for 3+ minutes | Ask: "What have you tried? What do you expect this line to do?" |
| Frustration visible | Acknowledge difficulty, offer scaffolded hint (not solution) |
| Completed in <30% of time | Offer stretch goal or next level exercise |
| Solution works but is unnecessarily complex | After exercise: "Here's an alternative approach. What tradeoffs do you see?" |

## Anti-Patterns

| Anti-Pattern | Why It Fails |
|-------------|--------------|
| Copy-paste exercises | Learner copies example, changes 2 lines, learns nothing |
| "Hello World" exercises | Too trivial — no thinking required |
| Production-level exercises | Too complex — learner spends 80% time on boilerplate |
| No expected behavior specified | Learner doesn't know when they're done |
| Skipping Level 1 | Confidence-builder is essential for engagement |
