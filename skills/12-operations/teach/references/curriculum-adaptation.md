# Curriculum Adaptation

## When to Adapt

The learning path created during Phase 2 is a hypothesis about what the learner needs. After 3+ sessions, you have data about what they ACTUALLY need. Adapt or the curriculum becomes a prison.

## Adaptation Triggers

### ACCELERATE (3+ consecutive 5/5 teach-backs)
- The learner is ahead of the curriculum
- Skip sessions for concepts the learner already demonstrates
- Merge related concepts if the learner connects them naturally
- Add challenge: optional advanced sessions if the learner wants depth

### DECELERATE (teach-back below 3/5)
- The current pace exceeds the learner's absorption rate
- Check: is a prerequisite concept missing? Add a session for it
- Break the current concept into 2-3 smaller sessions
- Add more guided practice before independent practice

### SIDEQUEST (learner shows strong interest in adjacent topic)
- The learner's curiosity is a learning accelerator — don't suppress it
- If the adjacent topic doesn't block the main curriculum, schedule as optional session
- If it does relate, weave it in as an application exercise

### DISENGAGEMENT (learner responses become minimal, no questions)
- Check if the goal is still relevant to the learner
- Check if the pace is wrong (too fast or too slow)
- Check for external factors (work pressure, life events)
- Consider: pause the curriculum, schedule a check-in session

## Adaptation Protocol

1. **Review last 3 sessions:** Teach-back ratings, exercise completion times, learner questions
2. **Identify pattern:** Accelerating, decelerating, or stable?
3. **Propose change:** "Based on the last 3 sessions, I suggest [change] because [evidence]."
4. **Get learner input:** "Does this match your experience? What would you change?"
5. **Update path:** Modify `.teach/learning-path.md`, record rationale for the change
6. **Continue:** Resume with adapted curriculum

## Adaptation Record

```markdown
## Curriculum Adaptations

### [YYYY-MM-DD] After Session 5
- Trigger: 3 consecutive teach-backs rated 2/5 on async concepts
- Change: Added prerequisite session on "Event Loop & Call Stack" before continuing with Promises
- Rationale: Learner was trying to understand Promises without understanding the execution model
- Outcome: [to be evaluated after +3 sessions]
```

## When NOT to Adapt

| Situation | Why Not |
|-----------|---------|
| Single bad session | Could be fatigue, external factors. Wait for 3-session pattern. |
| Learner says "this is too hard" | Productive struggle feels hard. Check teach-back before adapting. |
| Learner is bored with fundamentals | Fundamentals must be solid. Add challenge within the same concept, don't skip. |
| Curriculum is "almost done" (2 sessions left) | Adaptation overhead > benefit. Finish, then plan next steps. |
